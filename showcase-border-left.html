1. Info Cards

Basic icon + text cards. Four semantic color variants.

Problem

Training CNFs via maximum likelihood requires expensive ODE simulations, and diffusion processes restrict the space of probability paths, leading to slow training and sampling.

Motivation

A simulation-free training objective for CNFs would enable scalable generative modeling beyond the constraints of diffusion processes.

Method

Flow Matching (FM) regresses a neural network onto target vector fields of fixed conditional probability paths, enabling simulation-free CNF training with Gaussian or Optimal Transport paths.

Result

FM with OT paths achieves state-of-the-art FID of 20.9 on ImageNet 128x128 with faster training, better likelihood (NLL 2.99 on CIFAR-10), and efficient sampling.

3. Timeline

Sequential process displayed as a timeline.

Step 1

Define Probability Path

Choose a conditional probability path p_t(x|x_1) from noise p_0 to data-centered distribution p_1.

Step 2

Construct Vector Field

Derive the conditional vector field u_t(x|x_1) that generates the chosen probability path.

Step 3

Train via CFM Loss

Regress a neural network v_t onto u_t using the Conditional Flow Matching objective (simulation-free).

Step 4

Sample via ODE

Generate samples by solving the ODE with the learned vector field using off-the-shelf solvers like dopri5.

4. Accordion

Collapsible FAQ cards.

Why not just use score matching for CNFs?

Score matching is restricted to diffusion processes with curved, suboptimal paths. Flow Matching works with any probability path, including Optimal Transport paths that are simpler and more efficient.

How is FM simulation-free?

The Conditional Flow Matching (CFM) objective uses per-sample conditional vector fields that can be computed in closed form, avoiding expensive ODE simulations during training.

What makes OT paths better than diffusion paths?

OT paths form straight-line trajectories with constant speed, whereas diffusion paths are curved and can overshoot. This leads to faster training, lower NFE for sampling, and better generalization.

16. Terminal / Code Card

Terminal window style. Great for code and CLI commands.

flow_matching.py
# Conditional Flow Matching (OT path)
def cfm_loss(v_net, x1, sigma_min=1e-4):
t = torch.rand(x1.shape[0])
x0 = torch.randn_like(x1)
mu_t = t * x1 # OT mean
sigma_t = 1 - (1 - sigma_min) * t
x = mu_t + sigma_t * x0
target = x1 - (1 - sigma_min) * x0
return (v_net(t, x) - target).pow(2).mean()
